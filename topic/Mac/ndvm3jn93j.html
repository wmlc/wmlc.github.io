使用 fs_usage 查看 macOS 进程的文件访问行为

在 macOS 系统中，如果你想知道某个程序正在读写哪些文件（比如配置文件、缓存、日志等），有一个非常实用但鲜为人知的命令行工具：fs_usage。本文将通过一个真实案例，带你了解如何使用它来追踪进程的文件系统活动。

✅ fs_usage 是什么？

  • macOS 自带的实时文件系统监控工具。
  • 可监控所有进程或指定 PID的文件、网络、磁盘 I/O 活动。

✅ 用法

sudo fs_usage -f pathname <PID>  

**`-f pathname`** 是最推荐的过滤器，输出包含完整文件路径，可读性强。

用法：
fs_usage [-e] [-w] [-f 模式] [-b] [-t 秒数] [-R 原始文件 [-S 起始时间] [-E 结束时间]] [进程ID | 命令名 [进程ID | 命令名] ...]

选项说明：
  -e    从采样中排除指定的进程 ID 列表，并默认排除 fs_usage 自身
  -w    强制输出更宽、更详细的格式
  -f    根据指定的模式筛选输出内容：
          "network"   显示与网络相关的事件
          "filesys"   显示与文件系统相关的事件
          "pathname"  仅显示与路径名相关的事件
          "exec"      仅显示 exec 和 spawn（执行与派生）事件
          "diskio"    仅显示磁盘 I/O 事件
          "cachehit"  额外显示缓存命中（cache hit）事件
  -b    为磁盘 I/O 事件添加 BootCache 信息（如果可用）
  -t    指定超时时间（秒），适用于自动化工具
  -R    指定要处理的原始跟踪文件（raw trace file）
  -S    若使用了 -R，此选项指定跟踪的起始时间（单位：微秒）
  -E    若使用了 -R，此选项指定跟踪的结束时间（单位：微秒）

参数说明：
  pid   指定要采样的进程 ID
  cmd   指定要采样的命令名称（匹配进程命令行）

默认行为（不带任何选项时）：
以下进程将被自动排除在输出之外：
fs_usage、Terminal（终端）、telnetd、sshd、rlogind、tcsh、csh、sh

✅ 常见事件类型（在 pathname 模式下）

  • open / close：文件打开/关闭
  • stat64 / lstat64：获取文件元信息
  • readlink：读取符号链接目标

✅ 适用场景

  • 调试程序配置文件位置
  • 分析软件缓存/日志行为
  • 安全审计（查看敏感文件是否被访问）
  • 逆向工程（无需反编译，直接观察运行时行为）

🎯 场景示例：查找 Vercel CLI 的认证令牌位置

作者 Simon Willison 想知道 Vercel CLI 把它的认证令牌（token）存在哪里。他没有去翻文档，而是直接用系统工具“现场取证”。

第一步：启动目标进程

$ vercel login

Vercel CLI 会暂停并等待用户选择登录方式（比如 GitHub），此时进程仍在运行，为我们提供了追踪窗口。

第二步：获取进程 PID

在另一个终端窗口中，用 ps 查找进程 ID：

$ ps aux | grep vercel
simon 87587 ... node /opt/homebrew/bin/vercel login

记下 PID：87587

第三步：使用 fs_usage 监控文件访问

运行以下命令（需要 sudo 权限）：

sudo fs_usage -f pathname 87587
  • -f pathname：只显示与路径名相关的文件系统事件（如 open、stat64、close 等），输出清晰且包含完整路径。
  • 87587：要监控的进程 ID。

然后回到 Vercel 登录窗口，完成登录操作。fs_usage 会实时输出该进程访问的所有文件。

第四步：定位关键文件

输出中出现了关键路径：

08:02:07.011124  lstat64  /Users/simon/Library/Application Support/com.vercel.cli/auth.json
08:02:07.011716  open     /Users/simon/Library/Application Support/com.vercel.cli/auth.json.1284651536

这说明 Vercel 将认证信息写入了：

~/Library/Application Support/com.vercel.cli/auth.json

验证一下：

$ cat ~/Library/Application\ Support/com.vercel.cli/auth.json
{
  "// Note": "This is your Vercel credentials file. DO NOT SHARE!",
  "token": "..."
}

✅ 成功定位！

⚠️ 注意事项

  • 需要 sudo 权限（因为涉及系统级监控）。
  • 不加 PID 会输出全系统所有活动，信息量极大，慎用！
  • 输出是实时流式数据，建议配合 grep 或重定向保存分析。

🧩 对比其他工具

工具 用途 特点
fs_usage 实时监控文件/网络 I/O 系统级、需 root、支持按 PID 过滤
lsof 列出当前打开的文件 快照式，不实时
opensnoop (via dtrace) 跟踪文件打开事件 更轻量，但需启用 SIP 或使用 bpftrace

对于“我想知道这个程序刚才是不是读了某个文件？”这类问题，fs_usage -f pathname PID 往往是最直接的答案。

📌 结语

fs_usage 是 macOS 下一个强大但低调的调试利器。下次当你好奇某个 App 把数据藏在哪，或者怀疑程序行为异常时，不妨试试它——只需一行命令，真相就在眼前。

小技巧：搭配 grep 使用更高效，例如：

sudo fs_usage -f pathname 87587 | grep '\.json'

Linux 下可以使用 strace 命令 查看进程正在做什么。
