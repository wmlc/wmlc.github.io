华为云服务器(Ubuntu)配置多个公网 IP 访问

  • 1、添加网卡
  • 2、购买弹性公网IP
  • 3、将弹性公网 IP 绑定 弹性网卡
  • 4、为各网卡添加策略路由
  • 5、使配置永久生效

以下操作适用于 Ubuntu 服务器

1、添加网卡

通过华为云 ECS 控制台创建新的弹性网卡

创建成功后通过 ssh 登录服务器, 使用 ifconfig 命令查看可以发现新生成 eth1 网卡

2、购买弹性公网IP

通过华为云 ECS 控制台购买即可

3、将弹性公网 IP 绑定 弹性网卡

4、为各网卡添加策略路由

参考官方文档

# 验证内网IP和公网IP是否可以通信

ping -I {内网IP地址}  {外网IP地址}

# 依次执行以下命令，添加主网卡和扩展网卡的策略路由。绑定内网IP和公网IP之间的通信

# 主网卡
ip route add default via 192.168.0.1 dev eth0 table 10

ip route add 192.168.0.0/24 dev eth0 table 10

ip rule add from {内网IP} table 10

# 扩展网卡
ip route add default via 192.168.0.1 dev eth1 table 20

ip route add 192.168.0.0/24 dev eth1 table 20

ip rule add from {内网IP} table 20

上面命令中的 子网网关、网卡名称、路由表名称、子网网段、网卡地址 等是我这边的配置，配置时需要通过 ifconfig 命令查看自己的具体配置

配置完成后可以再通过 ping -I 命令查看是否绑定成功， 绑定成功后就可以通过公网 IP 访问这台服务器了。

5、使配置永久生效

执行以下命令，为systemd服务创建一个新的“network-routes.service”文件。

vim /etc/systemd/system/network-routes.service

[Unit]
Description=Network Routes Configuration
After=network.target

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/bin/bash -c 'for((x=0; x<10; x++)); do [[ $(ip addr show eth0 | grep -w 192.168.0.136 >/dev/null 2>&1 && echo 1) ]] && break; sleep 1; done; ip route flush table 10; ip route add default via 192.168.0.1 dev eth0 table 10; ip route add 192.168.0.0/24 dev eth0 table 10; ip rule add from 192.168.0.136 table 10; for((x=0; x<10; x++)); do [[ $(ip addr show eth1 | grep -w 192.168.0.174 >/dev/null 2>&1 && echo 1) ]] && break; sleep 1; done; ip route flush table 20; ip route add default via 192.168.0.1 dev eth1 table 20; ip route add 192.168.0.0/24 dev eth1 table 20; ip rule add from 192.168.0.174 table 20; ip rule add to 169.254.169.254 table main'

[Install]
WantedBy=multi-user.target

保存后，执行以下命令，重新加载systemd配置，并启动服务。

systemctl daemon-reload

systemctl enable network-routes.service

回显类似如下信息，表示启动成功。

root@ecs-s:~# systemctl daemon-reload 
root@ecs-s:~# systemctl enable network-routes.service
Created symlink /etc/systemd/system/multi-user.target.wants/network-routes.service → /etc/systemd/system/network-routes.service.

配置完成。
