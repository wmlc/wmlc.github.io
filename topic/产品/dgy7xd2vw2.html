🖼️ 常见图片格式一览表

图片格式主要分为 ​​位图（栅格图）​​ 和 ​​矢量图​​ 两大类，每种格式有不同的特点，适用于不同的场景。

位图由像素（Pixel）组成，放大后会失真（变模糊）。

矢量图由数学公式定义，可无限放大而不失真。

下面图片格式中除 svg 为矢量图外其他均为 位图。

常见的位图格式包括：

(1) JPEG/JPG（Joint Photographic Experts Group）
特点：有损压缩，文件较小，适合照片、网页图片。

优点：高压缩比，色彩丰富，兼容性好。

缺点：不支持透明背景，多次编辑会降低质量。

适用场景：摄影、网页图片、社交媒体。

(2) PNG（Portable Network Graphics）
特点：无损压缩，支持透明背景（Alpha 通道）。

优点：适合带透明度的图片（如 Logo、UI 元素），质量高。

缺点：文件比 JPEG 大，不适合高分辨率照片。

适用场景：网页设计、UI 图标、透明背景图片。

(3) GIF（Graphics Interchange Format）
特点：支持动画、透明背景（仅 1-bit 透明），最多 256 色。

优点：适合简单动画（如表情包、小图标）。

缺点：色彩少，不适合照片或复杂图形。

适用场景：短动画、表情包、低色彩 Logo。

(4) WebP
特点：Google 开发，支持有损/无损压缩、透明背景和动画。

优点：比 PNG 和 JPEG 更小，适合网页优化。

缺点：部分旧浏览器（如 IE）不支持。

适用场景：网页图片、替代 PNG/JPEG/GIF。

(5) TIFF（Tagged Image File Format）
特点：无损压缩，支持多图层、高质量存储。

优点：适合专业打印、存档。

缺点：文件极大，不适合网页。

适用场景：印刷、扫描文档、专业摄影。

(6) BMP（Bitmap）
特点：无压缩，文件极大，Windows 默认格式。

优点：兼容性好，无损存储。

缺点：占用空间大，不适合网络传输。

适用场景：Windows 系统存储、简单图形。

(7) HEIF/HEIC（High Efficiency Image Format）
特点：苹果推广的新格式，比 JPEG 更高效。

优点：高压缩比，支持透明度和动画。

缺点：兼容性较差（需 macOS/iOS 或新软件支持）。当前移动端都支持了，但PC浏览器还不支持。

适用场景：iPhone 照片存储。

常见矢量图格式包括：

(1) SVG（Scalable Vector Graphics）
特点：基于 XML 的矢量格式，支持动画和交互。

优点：文件小，可缩放，适合网页和 UI 设计。

缺点：不适合复杂照片。

适用场景：Logo、图标、网页图形。

(2) EPS（Encapsulated PostScript）
特点：印刷行业标准，支持矢量+位图混合。

优点：适合专业设计、打印。

缺点：需专业软件（如 Adobe Illustrator）编辑。

适用场景：印刷品、矢量设计。

(3) PDF（Portable Document Format）
特点：可包含矢量+位图，支持多页文档。

优点：通用性强，适合图文混合存储。

缺点：编辑需专业工具。

适用场景：文档、电子书、打印文件。

(4) AI（Adobe Illustrator）
特点：Adobe 专用矢量格式，支持高级设计功能。

优点：专业设计工具标准格式。

缺点：需 Adobe Illustrator 打开。

适用场景：专业平面设计。

🔍 软件开发中的差异维度

1. ✅ 文件大小（性能/加载速度）

格式 描述
WebP / AVIF / HEIC 极致压缩，适合移动端和网络加载
JPG 有损压缩，最常见的照片格式，体积小
PNG / GIF 体积较大，适合小图或透明图层
SVG 非位图，文件小但依赖渲染器，适合图标/图形UI

2. 🖌️ 是否支持透明通道（Alpha）

| 需要透明效果的图像（如头像、icon、贴图等） |

格式 透明支持
PNG, WebP, AVIF, HEIC, SVG ✅ 支持
JPG, BMP ❌ 不支持

3. 🔄 是否支持多帧 / 动图（动画）

格式 动图支持 帧控制
GIF ✅ 有限支持 ✅
WebP / AVIF / HEIC ✅ 支持更强 ✅
PNG / JPG / BMP ❌ ❌

💡 开发者重点注意：

  • GIF 兼容广，但仅支持 256 色
  • WebP 动图 是现代替代，需浏览器支持(部分旧浏览器 如 IE 不支持), 比 GIF 更小、更清晰
  • AVIF 动图 支持更强但兼容性差

4. 🧱 平台/浏览器兼容性

格式 浏览器/平台支持情况
JPG / PNG / GIF ✅ 所有平台全面支持
WebP ✅ 现代浏览器已全面支持
AVIF 🟡 部分新浏览器支持，老设备需降级
HEIC/HEIF ❌ 仅 macOS/iOS 原生支持；Web端支持差
SVG ✅ 所有现代浏览器支持（矢量图）

5. 🛠️ 图像处理（开发工具兼容性）

格式 Imagick 支持 GD 支持 Intervention
JPG / PNG / GIF ✅ 广泛支持 ✅ ✅
WebP ✅ 需要 ImageMagick + libwebp 🟡（部分版本） ✅（需 Imagick 驱动）
AVIF / HEIC 🟡 需手动编译 libavif / libheif ❌ 🟡 部分不支持

💡 开发者常见选择建议

场景 推荐格式 原因
用户头像、图标 PNG / WebP 支持透明、兼容性好
移动端照片上传 JPEG / HEIC 压缩小、速度快
网站图像加速 WebP / AVIF 文件小，加载快
Logo / 矢量图（UI/图标） SVG / EPS / AI 矢量缩放不失真
动图 WebP 动图（优先） / GIF（兼容）/ APNG 更小、支持多帧控制
网页图片 WebP > PNG > JPEG
透明背景图片 PNG / WebP
摄影/照片 JPEG / HEIC / WebP
打印/高质量存档 TIFF / PDF / EPS

✅ 最后的建议

在软件开发中，不应仅考虑图片是否“看起来对”，更应根据用途、性能、兼容性等因素综合选择格式：

  • ✅ 前端要考虑浏览器支持
  • ✅ 后端处理要关注库支持（如 Imagick 是否编译支持 WebP）
  • ✅ 移动端开发要考虑平台原生支持（如 iOS 用 HEIC）
  • ✅ 生成动图、裁图、转码建议使用 ImageMagick/Imagick，掌握底层能力
