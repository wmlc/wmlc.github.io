最近遇到了一个项目的 Git 仓库太大了导致无法提交到码云的远程 git 仓库中，因此想到可以通过删除某些大的或已不再使用的文件的历史提交纪录来缩小仓库。操作步骤：

1、查找仓库中的大文件

# 列出前十个最大文件

git rev-list --objects --all |   git cat-file --batch-check='%(objecttype) %(objectsize) %(rest)' |   grep '^blob' |   sort -k2 -n |   tail -10 |   awk '{printf "%.2f MB\t%s\n", $2/1024/1024, $3}'

# 输出
8.80 MB tool_scripts/app_distribution
9.30 MB tool_scripts/app_distribution.exe

找到大文件后就可以考虑是否进行删除，若大文件在最新版本中已经删除可以直接清理；若大文件还在使用可以清理后将大文件的引入改为 Git LFS（Large File Storage）方案或「外部文件存储 + 自动下载」的方案。

2、清理 Git 历史

2.1、安装 git-filter-repo

# macOS (Homebrew)
brew install git-filter-repo

# 或者通过 Python 安装
pip install git-filter-repo

2.2、验证仓库大小

清理前后执行以下命令，查看当前仓库大小，验证清理是否生效

git count-objects -vH

2.3、执行删除目录历史的命令

前提其他用户那边所有修改已提交，执行命令的用户拉取最新的代码。

# 删除目录 secret_folder/
git filter-repo --path secret_folder/ --invert-paths

# 删除文件 os/libeffect-sdk.a
git filter-repo --path ios/libeffect-sdk.a --invert-paths

# 解释：

--path secret_folder/：指定要处理的路径

--invert-paths：表示“删除这些路径，而不是保留它们”

这个命令会：

  • 从所有提交中移除该目录及其文件

  • 重写整个历史

  • 让该目录看起来从未存在过

2.4、重新添加远程仓库

git filter-repo 默认会清理掉旧的远程配置，以防误推改写历史, 因此要想将处理后的内容推送到远程仓库需要重新添加远程仓库地址。

git remote add origin {远程仓库地址}

注意：若清理后明明本地的已明显缩小，但还是推送不到原git地址，提示仓库太大，可以新建仓库，将远程仓库地址改为新的仓库进行推送，后续就用这个新的仓库了

2.5、强制推送回远程

git filter-repo 执行过程中会拉取所有分支，全部处理， 因此执行完成后直接推送全部分支标签即可

# 注意要推送全部分支和标签

git push origin --force --all

git push origin --force --tags

警告：这会重写远程仓库的历史！其他协作者需要重新克隆仓库。

2.6、命令整合成bash脚本

#!/bin/bash

# 执行该脚本清理大文件历史，缩小仓库体积

if [[ $(git status --porcelain | wc -l) -gt 0 ]]; then
  echo "你有未添加的修改，请先添加。"
  exit 1
fi

git count-objects -vH

git filter-repo --path {要删除的文件} --invert-paths

git remote add origin {远程仓库地址}

git push origin --force --all

git push origin --force --tags

git count-objects -vH

2.7 只清除某个分支场景

【测试无效，暂不考虑】

以上步骤中是清除所有分支中的某个文件或目录，此时所有分支、标签的历史都被重写，commit hash 全部改变。

其他用户那边本地仓库与远程仓库完全“脱节”，因此其他用户那边需要完全删除仓库，重新 clone。

但有时我们确定要删除的文件只存在于某个分支中，可以执行命令只删除这个分支上的文件，这样其他用户不需要删除仓库重新克隆，只需要pull下指定分支即可。

前提：要保证其他用户那边这个分支的修改已提交再做处理

# 重写指定分支历史
git filter-repo --path secret_folder/ --invert-paths --refs refs/heads/{分支名}

git push -f

// 其他用户那边

git pull --force

git reset --hard origin/{分支名}

3、避免git仓库的快速膨胀

3.1、大文件问题

Git 本质上是快照存储系统，每次提交都会保存文件的完整版本（即使只改了一点点）。

因此：

频繁更新的大文件会快速膨胀仓库体积。

即使后来删除了文件，历史中仍保留副本。

平台（如 码云）通常限制仓库 ≤ 1GB，单文件 ≤ 100MB。

因此若将大文件放入 git 中，很快就占满了。

3.2、方案一：Git LFS

🧠 原理

Git LFS 把大文件（如 .mp4, .zip, .psd, .model）替换为一个「小指针文件」，而真正的大文件被存放到一个独立的 LFS 服务端（可自建或托管）。

Git 版本历史只记录指针，不再保存文件本体。拉取时 Git 自动下载对应的 LFS 文件。

因为用的远程服务 Git LFS 空间也有限，未使用该方案，此处不做具体介绍。

3.3、方案二：外部文件存储 + 自动下载

在仓库中只保留文件的下载链接（如从 OSS/CDN 下载）：

/assets/
  model_v1.txt  # 内容：s3://my-bucket/models/model_v1.zip

配合脚本在构建时自动下载：

python3 scripts/download_assets.py

✅ 优点：

无需改 Git 配置

仓库超轻量

可复用对象存储（如阿里云 OSS、AWS S3、七牛云）, 空间无限量

具体实现

1、目录结构示例

project/
├── assets/download/ # 脚本会自动扫描该目录下的 txt 文件
│   ├── model_v1.txt        # 存放模型下载地址（或元信息）
│   ├── texture_pack.txt    # 存放大纹理包链接
│
├── scripts/
│   └── download_assets.py  # 下载脚本
│
├── .gitignore
├── README.md
└── ...

2、assets 文件示例

assets/download/models.txt

# 要下载到本地的远程文件
# 格式: <目标子目录> <URL> [unzip]
models https://cdn.example.com/models/model_v1.zip unzip
models https://cdn.example.com/models/config.json
textures https://cdn.example.com/textures/wood.zip unzip

3、download_assets.py 脚本

新建：scripts/download_assets.py

#!/usr/bin/env python3
"""
下载 assets/download/*.txt 中定义的资源文件，并支持可选解压与清理。有 unzip 标识需要解压

格式:
<目标目录> <URL> [unzip]

例如:
models https://cdn.example.com/models/model_v1.zip unzip
textures https://cdn.example.com/textures/wood.png
"""

import os
import sys
import shutil
import requests
import zipfile
import tarfile
from pathlib import Path
from concurrent.futures import ThreadPoolExecutor

# === 配置 ===
ROOT_DIR = Path(__file__).resolve().parent.parent
ASSETS_DIR = ROOT_DIR / "assets/download"
DOWNLOAD_BASE = ROOT_DIR
MAX_WORKERS = 4

def download_file(url: str, target_path: Path):
    """下载单个文件，带断点续传"""
    target_path.parent.mkdir(parents=True, exist_ok=True)
    headers = {}
    if target_path.exists():
        local_size = target_path.stat().st_size
        headers["Range"] = f"bytes={local_size}-"
    else:
        local_size = 0

    try:
        r = requests.get(url, stream=True, headers=headers, timeout=30)
        r.raise_for_status()
    except Exception as e:
        print(f"❌ 下载失败: {url} ({e})")
        return False

    total_size = int(r.headers.get("Content-Length", 0)) + local_size
    mode = "ab" if local_size > 0 else "wb"

    with open(target_path, mode) as f:
        for chunk in r.iter_content(chunk_size=8192):
            if chunk:
                f.write(chunk)

    print(f"✅ 下载完成: {target_path.relative_to(ROOT_DIR)} ({total_size/1024:.1f} KB)")
    return True

def cleanup_macosx_files(dest_dir: Path):
    """删除 __MACOSX 文件夹"""
    for root, dirs, _ in os.walk(dest_dir):
        for d in dirs:
            if d == "__MACOSX":
                mac_path = Path(root) / d
                try:
                    shutil.rmtree(mac_path)
                    print(f"🧹 已删除冗余目录: {mac_path.relative_to(ROOT_DIR)}")
                except Exception as e:
                    print(f"⚠️ 删除 __MACOSX 失败: {e}")

def extract_file(file_path: Path, dest_dir: Path):
    """根据文件类型自动解压并清理"""
    try:
        if zipfile.is_zipfile(file_path):
            with zipfile.ZipFile(file_path, "r") as zf:
                zf.extractall(dest_dir)
            print(f"📦 已解压 ZIP: {file_path.relative_to(ROOT_DIR)} → {dest_dir.relative_to(ROOT_DIR)}")
        elif tarfile.is_tarfile(file_path):
            with tarfile.open(file_path, "r:*") as tf:
                tf.extractall(dest_dir)
            print(f"📦 已解压 TAR: {file_path.relative_to(ROOT_DIR)} → {dest_dir.relative_to(ROOT_DIR)}")
        else:
            print(f"⚠️ 文件不是压缩包: {file_path.name}")
            return

        # 清理压缩包
        try:
            file_path.unlink()
            print(f"🗑️ 已删除压缩包: {file_path.relative_to(ROOT_DIR)}")
        except Exception as e:
            print(f"⚠️ 删除压缩包失败: {e}")

        # 清理 __MACOSX
        cleanup_macosx_files(dest_dir)

    except Exception as e:
        print(f"❌ 解压失败: {file_path.name} ({e})")

def process_meta_file(meta_path: Path, tasks: list):
    """解析 assets 文件"""
    print(f"📦 处理资源清单: {meta_path.relative_to(ROOT_DIR)}")
    with open(meta_path, "r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if not line or line.startswith("/topic/Git/3dykq7zyl0.html#"):
                continue

            parts = line.split()
            if len(parts) < 2:
                print(f"⚠️ 格式错误: {line}")
                continue

            target_dir, url = parts[:2]
            need_unzip = len(parts) >= 3 and parts[2].lower() == "unzip"
            filename = os.path.basename(url)
            target_path = DOWNLOAD_BASE / target_dir / filename
            tasks.append((url, target_path, need_unzip))

def main():
    DOWNLOAD_BASE.mkdir(exist_ok=True)

    meta_files = list(ASSETS_DIR.glob("*.txt"))
    if not meta_files:
        print("❌ 未找到任何 assets/*.txt 文件")
        sys.exit(1)

    download_tasks = []
    for meta in meta_files:
        process_meta_file(meta, download_tasks)

    if not download_tasks:
        print("✨ 无下载任务。")
        return

    print(f"⬇️ 共需下载 {len(download_tasks)} 个文件...")

    def worker(task):
        url, target_path, need_unzip = task
        if target_path.exists():
            print(f"✅ 已存在: {target_path.relative_to(ROOT_DIR)}")
        else:
            ok = download_file(url, target_path)
            if not ok:
                return
        if need_unzip:
            extract_file(target_path, target_path.parent)

    with ThreadPoolExecutor(max_workers=MAX_WORKERS) as executor:
        executor.map(worker, download_tasks)

    print(f"\n✅ 所有任务完成，资源已保存至 {DOWNLOAD_BASE.relative_to(ROOT_DIR)}/")

if __name__ == "__main__":
    main()
